1. 프레임워크란?
    - 특정 운영체제를 위한 응용프로그램 표준 구조를 구현하는 클래스와 라이브러리의 모임
    - 서버 프레임워크는 다양한 네트워크 설정, 요청 및 응답 처리를 손쉽게 처리할 수 있도록
       클래스나 인터페이스를 제공하는 소프트웨어를 의미


2. Array와 List
    1) 공통점
        - Array와 List는 목록, 순서가 있는 일련의 집합체
    2) 차이점
        - Array는 index자리에 value가 삭제되면 뻥 떠버리기 때문에 메우기위한 로직이 필요
        - List는 index자리에 value가 삭제되면 index를 앞으로 전진시킴

3. Map
    - 객체와 비슷하지만 메소드만을 이용해 값을 넣고 뺌.
    EX) let max = new Map();
        max.set("id", 0);
        max.set("이름", "마이클");

        let michael = new Map([
            ["id", 0],
            ["이름", "마이클"],
        ]);

        max.get("이름);  => "마이클"
        max.delete("나이) => 삭제 성공시 true 반환
        max.clear();    => 맵 안의 프로퍼티 전부삭제


4. 자바스크립트의 장점
    - 인터프리터 형식이기 때문에 컴파일 과정이 없다
    - C나 자바와 달리 굉장히 단순한 구조와 원칙을 가지고 있어서 쉬움
        EX) int, string을 var나 let으로 통일가능함(인터프리터 형식이기 때문)

5. 인터프리터와 컴파일
    1) 인터프리터
        - 소스코드를 바로 실행하는 컴퓨터 프로그램 또는 환경
    2) 컴파일러
        - 원시코드를 기계어로 번역

6. 쿠키와 세션
    - 클라이언트와 서버의 통신이 끝나면 그 정보는 유지가 되지 않기 때문에 쿠키와 세션을 사용을 하여 저장한다.
    1) 쿠키
        - 클라이언트에 상태정보를 저장
        - 세션 쿠키와 지속 쿠키로 나뉜다.
        - 세션쿠키는 브라우저가 종료되면 정보를 삭제, 
          지속 쿠키는 만료날짜/시간을 설정하여 그 시간이 지나면 정보를 삭제
    2) 세션
        - 서버에 상태정보를 저장하고 클라이언트에게 세션ID를 부여하여 구분을 함

7. get방식과 post방식
    - HTTP 프로토콜에서 서버에게 요청을 보내는 메서드
    1) GET
        - 필요한 데이터를 쿼리스트링에 담아 전송하기 떄문에 보안 bad
        - ?나 &로 요청
        - 단점)
            - 불필요한 요청을 제한하기 위해 캐시에 저장되는데, 
              캐시를 지우지않으면 컨텐츠를 변경해도 바뀌지 않는 경우가 발생할 수 있음
            - 대용량 데이터 요청에 불편

    2) POST
        - 리소스를 생성/변경하기 위해 설계된 메서드
        - Body에 담아 요청하기 때문에 보안 good
        - 대용량 데이터 요청에 용이
